# Frontend Test Assignment — Polymarket Clone

## 1. Цель

Реализовать приложение на Next.js, повторяющее ключевой UX Polymarket.

### Основные требования
- Точное воспроизведение интерфейса
- Live-обновление цен
- Чистое управление состоянием

---

## 2. Технологический стек

### Обязательные технологии
- Next.js (App Router)
- TypeScript
- Jotai (state management)

### Ограничения
- Запрещено использовать Redux

### API
- https://docs.polymarket.com/api-reference
- https://gamma-api.polymarket.com/events?closed=false

---

## 3. Доменная модель

Polymarket — prediction market:

- Пользователи торгуют исходами событий
- Цена (0–1) = вероятность события

### Требование к UI
- Цена должна визуально отражать вероятность

---

## 4. Функциональные требования

### 4.1 Главная страница (Events List)

#### Требования
- Grid или список карточек
- Карточка события содержит:
    - title
    - volume
    - preview рынка (опционально)

---

### 4.2 Навигация по категориям

#### Требования
- Верхнее меню:
    - Crypto
    - Sports
    - Politics
- Клик по категории:
    - фильтрация списка событий

#### Ограничения
- Допускается клиентская фильтрация

---

### 4.3 Страница события (Event Details)

#### Требования
- Заголовок события
- Список рынков (outcomes)

#### Каждый outcome:
- название
- цена
- probability bar

#### Важно
- Верстка максимально близка к Polymarket

---

## 5. Real-time обновления (критично)

### Требования
- Цены обновляются в реальном времени
- Без перезагрузки страницы

### Реализация
- WebSocket или симуляция

### Ограничения
- Нельзя перерендеривать всё дерево компонентов
- Обновления должны быть плавными

---

## 6. UI / UX

### Требования
- Визуально близко к Polymarket
- Четкая иерархия
- Корректные отступы
- Loading states

### Бонус
- Подсветка изменения цены
- Анимации
